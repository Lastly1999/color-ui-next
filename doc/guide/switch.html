# 🔰 Switch 开关

TIP

用来和用户交互是否启用禁用的选项组件

🔥 普通开关 注：将事件订阅出来，而不直接在内部改变checked的状态，是为了能更好的方便用户处理改变的事件，这种方式更加灵活

<template>
  <cu-switch @change="change" :checked="checked"></cu-switch>
</template>

export default{
    data(){
        return{
            checked:false
        }
    },
    methods:{
        change(){
            this.checked = !this.checked
        }  
    }
}

# 🎨 自定义开关颜色

🔥 自定义开关颜色，需要定义开关的颜色，需在属性上绑定 color="" 具体颜色可参考颜色表

<template>
  <cu-switch @change="change" :checked="checked" color="red"></cu-switch>
</template>

export default{
    data(){
        return{
            checked:false
        }
    },
    methods:{
        change(){
            this.checked = !this.checked
        }  
    }
}

# 🎁 自定义开关显示图标

🔥 自定义开关图标，目前仅支持 sex 性别的切换图标，直接在属性绑定 icon="sex"，后期会加入更多icon

<template>
  <cu-switch icon="sex" @change="change" :checked="checked" color="red"></cu-switch>
</template>

export default{
    data(){
        return{
            checked:false
        }
    },
    methods:{
        change(){
            this.checked = !this.checked
        }  
    }
}

# 📐 自定义开关形状

🔥 自定义开关形状，目前仅支持 radius 扁平的开关样式，直接在属性配置radius 即可转换为radius形状的开关

<template>
  <cu-switch radius @change="change" :checked="checked" color="red"></cu-switch>
</template>

export default{
    data(){
        return{
            checked:false
        }
    },
    methods:{
        change(){
            this.checked = !this.checked
        }  
    }
}

# 🔨 switch可选配置的参数

属性 可选参数 类型 说明
checked true/false boolean 开关启用状态
radius true/false boolean 是否为扁平形状
color 参考颜色表 string 按钮的颜色
